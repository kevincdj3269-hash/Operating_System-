優先權排程 (不可搶先式)

定義： 每個程序都有一個優先權等級（通常數字越小優先權越高）。CPU 總是選擇就緒佇列中最高優先權的程序執行，且一旦開始就不會被中斷。

💡 Step-by-Step 計算說明：
1. 0ms: 只有 P1 到達，直接開始執行。
2. 1~2ms: P2, P3 雖然優先權更高，但 P1 是不可搶先的，必須等到 5ms。
3. 5ms: P1 結束。比較 P2(Pri:1) 與 P3(Pri:2)，選擇 P2 先做。
4. 8ms: P2 結束，換 P3 執行到 10ms。
平均等待時間 (Avg WT): [(5-5-0) + (5-1) + (8-2)] / 3 = 3.33ms
✅ 優點
  • 符合現實需求（如 VIP 優先）。
  • 比搶先式更穩定，減少切換開銷。
❌ 缺點
  • 飢餓現象 (Starvation)：低優先權程序可能永遠等不到 CPU。
  • 解決方案：老化技術 (Aging)，隨時間增加等待程序的優先權。

Priority Scheduling (Non-preemptive)

Definition: CPU is allocated to the process with the highest priority. In non-preemptive mode, the running process cannot be interrupted.

✅ Pros: Honors critical tasks; lower overhead than preemptive.
❌ Cons: Starvation for low-priority tasks. Use Aging to fix.
