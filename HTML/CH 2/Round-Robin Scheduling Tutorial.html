📝 Step-by-Step 計算步驟：
1. 0-4ms: P1 執行 4ms，剩下 6ms，回到隊尾。
2. 4-8ms: P2 執行 4ms，剛好完成。
3. 8-10ms: P3 執行 2ms，完成（不滿 4ms 也會提早結束）。
4. 10-16ms: P1 回來分兩次把剩下的做完。
平均等待時間 (Avg WT):
P1: (0 + (10-4)) = 6ms | P2: 4ms | P3: 8ms
平均 WT = (6+4+8)/3 = 6ms

Round-Robin Scheduling (RR)

📝 Step-by-Step Calculation:
- P1 finishes at 16ms. TAT = 16-0 = 16. WT = 16-10 = 6ms.
- P2 finishes at 8ms. TAT = 8-0 = 8. WT = 8-4 = 4ms.
- P3 finishes at 10ms. TAT = 10-0 = 10. WT = 10-2 = 8ms.
Average WT = 6ms
✅ 優點 (Pros): 公平、響應快、無飢餓。 / Fair, fast response, no starvation.
❌ 缺點 (Cons): Context Switch 開銷大。 / High context switch overhead.
