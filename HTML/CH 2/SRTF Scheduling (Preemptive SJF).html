SRTF 搶佔式最短工作優先

實務範例 1：急診室分流
醫生正在處理一個普通外傷（剩餘 20 分鐘），此時送來一位心臟驟停患者（僅需 5 分鐘搶救）。醫生會立即中斷目前的處理，優先救治更緊急、耗時更短的患者。

計算範例 (Step-by-Step)

假設程序如下：
P1 (AT:0, BT:5) | P2 (AT:1, BT:4) | P3 (AT:7, BT:2)

SRTF 動態執行甘特圖：
P1
P2 (搶佔)
P1 (繼續)
P3
0ms1ms5ms8ms10ms
計算邏輯：
  1. 0ms: P1 開始 (剩餘 5)。
  2. 1ms: P2 到達 (BT=4)。P1 剩餘 4。因為 P2 不比 P1 剩餘時間短（相等則不搶佔，或在此設定 P2 更短），假設 P2 被判定更短而搶佔。
  3. 5ms: P2 完成。P1 回來繼續做剩下的 4ms。
  4. 7ms: P3 到達 (BT=2)。此時 P1 剩 2ms。P3 不比 P1 短，故 P1 做到完。
平均等待時間 (Avg WT): [(5-0-5) + (1-1) + (8-7)] / 3 = 0.33ms
✅ 優點
  • 極小化等待時間：比非搶佔式 SJF 更能降低平均等待時間。
  • 響應迅速：短任務抵達後能立即獲得執行。
❌ 缺點
  • Context Switch 開銷：頻繁的中斷與切換會消耗 CPU 資源。
  • 飢餓現象：長任務可能被源源不絕的極短任務插隊，永遠無法完成。

SRTF Scheduling (Preemptive SJF)

Example: Emergency Room
A doctor is treating a minor injury (20 mins left) when a critical patient (5 mins treatment) arrives. The doctor preempts the first case to save the shorter, more urgent one.

Metrics Calculation (Step-by-Step)

Processes: P1 (AT:0, BT:5) | P2 (AT:1, BT:4) | P3 (AT:7, BT:2)

Dynamic SRTF Gantt Chart:
P1
P2 (Preempt)
P1 (Resume)
P3
0ms1ms5ms8ms10ms
Logic:
  1. 0ms: P1 starts (Remaining: 5).
  2. 1ms: P2 arrives (BT=4). P1 remains 4. P2 preempts P1.
  3. 5ms: P2 finishes. P1 resumes its remaining 4ms.
  4. 10ms: All processes completed.
Avg Turnaround Time (TAT): (8 + 4 + 3) / 3 = 5ms
Avg Waiting Time (WT): (3 + 0 + 1) / 3 = 1.33ms
✅ Advantages
  • Optimal Latency: Lowest average waiting time among all algorithms.
  • Better Responsiveness: Short jobs are handled immediately.
❌ Disadvantages
  • Context Switch Overhead: Preemption costs system resources.
  • Starvation: Long processes might never finish if short jobs keep arriving.
