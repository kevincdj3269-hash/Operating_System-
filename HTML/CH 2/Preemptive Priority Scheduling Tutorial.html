優先權排程 (可搶先式 Preemptive)

核心機制： 當一個優先權較高的程序抵達時，系統會暫停當前運行的程序，並將 CPU 分配給新抵達的程序。

📊 Step-by-Step 計算步驟：
1. 0ms: P1 開始執行。
2. 2ms: P2 到達 (Pri:1)。因為 1 < 3，P2 搶走 CPU，P1 被迫中斷。
3. 6ms: P2 執行 4ms 完工。P1 回來繼續做剩下的 8ms。
4. 14ms: P1 最終完成。
平均等待時間 (Avg WT): [(14-10-0) + (6-4-2)] / 2 = (4 + 0) / 2 = 2ms
✅ 優點
  • 響應極快：高優先權任務抵達後幾乎立即執行。
  • 能有效處理系統緊急事件。
❌ 缺點
  • 飢餓現象 (Starvation)：若高優先權任務不斷，低優先權者永遠被暫停。
  • 切換成本：頻繁的 Context Switch 會降低效能。

Priority Scheduling (Preemptive)

Mechanism: If a new process arrives with a higher priority than the currently running process, the CPU is preempted (interrupted) and allocated to the newcomer.

📊 Metrics Calculation:
- P1: Finished at 14ms. TAT = 14 - 0 = 14. WT = 14 - 10 = 4ms.
- P2: Finished at 6ms. TAT = 6 - 2 = 4. WT = 4 - 4 = 0ms.
Average WT = 2ms
✅ Pros: Instant response for critical tasks; ideal for real-time systems.
❌ Cons: Starvation for low-priority tasks; high context switch overhead.
