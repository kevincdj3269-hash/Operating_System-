多層佇列排程 (Multilevel Queue)

場景： 系統將程序分為「高優先權佇列」(Q1) 與「低優先權佇列」(Q2)。只有當 Q1 為空時，CPU 才會處理 Q2。

P1 (Q1)
P2 (Q1)
P3 (Q2)
0ms3ms6ms10ms
📝 Step-by-Step 排程邏輯：
1. 0-6ms: CPU 發現 Q1 有任務 P1, P2，按 FCFS 順序執行。
2. 6ms: Q1 任務清空。
3. 6-10ms: CPU 終於可以處理 Q2 中的 P3。
計算平均等待時間 (假設 AT 皆為 0)：
P1: 0ms | P2: 3ms | P3: 6ms
平均等待時間 = (0+3+6)/3 = 3ms

Multilevel Queue Scheduling (MLQ)

Scenario: Processes are split into Q1 (High) and Q2 (Low). CPU handles Q2 ONLY when Q1 is empty.

P1 (Q1)
P2 (Q1)
P3 (Q2)
0ms3ms6ms10ms
📝 Metrics Calculation:
- Avg Turnaround Time (TAT) = (3+6+10)/3 = 6.33ms
- Avg Waiting Time (WT) = (0+3+6)/3 = 3ms
✅ 優點 (Pros): 為不同程序提供特製化排程。 / Tailored scheduling for different types of tasks.
❌ 缺點 (Cons): 飢餓現象 (Starvation)。 / Fixed queues lead to potential starvation.
