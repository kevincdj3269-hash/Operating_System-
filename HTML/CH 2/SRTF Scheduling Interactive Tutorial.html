計算步驟：
1. 0ms: P1 開始執行 (剩餘 8)。
2. 2ms: P2 到達 (BT:4)。此時 P1 剩 6。因為 4 < 6，P2 搶佔 P1。
3. 6ms: P2 完成。P1 回來繼續做剩下的 6ms。
4. 12ms: P1 完成。P3(BT:2) 在 5ms 抵達，此刻開始執行直到 14ms。

平均數據計算

周轉時間 (TAT) = 完成時間 - 到達時間
等待時間 (WT) = TAT - 執行時間 (BT)

  • P1: TAT = 12-0 = 12 | WT = 12-8 = 4
  • P2: TAT = 6-2 = 4 | WT = 4-4 = 0
  • P3: TAT = 14-5 = 9 | WT = 9-2 = 7

平均 WT = (4+0+7)/3 = 3.67ms

✅ 優點：極小化平均等待時間，適合短任務。
❌ 缺點：長任務可能飢餓，切換開銷大。

SRT Preemptive Scheduling

Calculation Steps:
1. 0ms: P1 starts (Remaining: 8).
2. 2ms: P2 arrives (BT:4). P1 remains 6. Since 4 < 6, P2 preempts P1.
3. 6ms: P2 finishes. P1 resumes its remaining 6ms.
4. 12ms: P1 finishes. P3 starts and runs until 14ms.

Metrics Results

  • Avg Turnaround Time (TAT) = 8.33ms
  • Avg Waiting Time (WT) = 3.67ms
✅ Pros: Optimal average waiting time; fast for short jobs.
❌ Cons: Possible starvation; high context switch overhead.
