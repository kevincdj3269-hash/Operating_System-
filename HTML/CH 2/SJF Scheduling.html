SJF 最短工作優先：進階計算與分析

1. 計算公式

完成時間 (CT) = 程序結束的時間點
周轉時間 (TAT) = CT - 到達時間 (AT)
等待時間 (WT) = TAT - 執行時間 (BT)

2. 實務範例：非搶佔式 SJF

當 P1 正在執行時，P2 與 P3 陸續到達。當 P1 完成後，SJF 會比較 P2 與 P3 的執行時間，選擇較短者先做。

程序 到達時間 (AT) 執行時間 (BT)
P10ms6ms
P22ms8ms
P34ms2ms
SJF 甘特圖執行順序 (P1 → P3 → P2)：
P1 (6ms)
P3 (2ms)
P2 (8ms)
0ms6ms8ms16ms
分步計算步驟：
1. P1: 0ms 開始，6ms 完成。 (TAT: 6-0=6, WT: 6-6=0)
2. 決策點 (6ms): 此時 P2(BT=8) 與 P3(BT=2) 都在等。SJF 選擇較短的 P3。
3. P3: 6ms 開始，8ms 完成。 (TAT: 8-4=4, WT: 4-2=2)
4. P2: 8ms 開始，16ms 完成。(TAT: 16-2=14, WT: 14-8=6)
平均周轉時間 (Avg TAT): (6 + 14 + 4) / 3 = 8ms
平均等待時間 (Avg WT): (0 + 6 + 2) / 3 = 2.67ms
✅ 優點 (Advantages)
  • 最低等待時間：理論上能達成最小的平均等待時間。
  • 效率高：快速清空短任務，提升系統吞吐量。
❌ 缺點 (Disadvantages)
  • 飢餓現象 (Starvation)：長程序可能被源源不絕的短程序無限期延後。
  • 不可知性：作業系統很難精準預測下一個任務到底要跑多久。

SJF Scheduling: Metrics & Analysis

1. Core Formulas

Completion Time (CT) = Time when process finishes
Turnaround Time (TAT) = CT - Arrival Time (AT)
Waiting Time (WT) = TAT - Burst Time (BT)

2. Case Study: Non-preemptive SJF

While P1 is running, P2 and P3 arrive. At 6ms, SJF compares P2 and P3, picking the shortest one.

Process Arrival (AT) Burst (BT)
P10ms6ms
P22ms8ms
P34ms2ms
SJF Execution Order (P1 → P3 → P2):
P1 (6ms)
P3 (2ms)
P2 (8ms)
0ms6ms8ms16ms
Step-by-Step Calculation:
1. P1: Starts at 0, finishes at 6. (TAT: 6-0=6, WT: 0)
2. Decision (6ms): P2(BT=8) and P3(BT=2) are ready. SJF selects P3.
3. P3: Starts at 6, finishes at 8. (TAT: 8-4=4, WT: 2)
4. P2: Starts at 8, finishes at 16. (TAT: 16-2=14, WT: 6)
Average TAT: (6 + 14 + 4) / 3 = 8ms
Average WT: (0 + 6 + 2) / 3 = 2.67ms
✅ Advantages
  • Optimal: Minimum average waiting time for any set of processes.
  • High Throughput: Clears short jobs quickly.
❌ Disadvantages
  • Starvation: Long processes may wait forever.
  • Estimation: Difficult to know actual burst time in advance.
